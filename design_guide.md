# Guideline: How to Design Anti-Aesthetic Websites

## 1. Start with intent, not ugliness

The first rule is: **anti-aesthetic design must have a reason**.

A bad anti-aesthetic website is just messy, unreadable, or annoying. A good anti-aesthetic website uses discomfort, awkwardness, decay, chaos, or visual wrongness to support a message, identity, critique, or emotional state.

Before touching layout or colors, define:

**What is this website resisting?**

For example:

* The polished sameness of SaaS websites
* The “clean lifestyle” look of wellness brands
* Corporate optimism
* Algorithmic beauty
* Minimalist luxury
* Instagram-perfect visual culture
* Smooth, frictionless consumer design
* Techno-utopian interfaces
* Sanitized emotional tone

Then define:

**What should the user feel?**

Possible answers:

* Unease
* Confusion
* Nostalgia
* Alienation
* Playfulness
* Distrust
* Anger
* Boredom
* Absurdity
* Grief
* Digital decay
* Low-budget sincerity
* Handmade imperfection

The paper argues that negative, strange, or “undesirable” aesthetics are valid when they match user intent and expressive purpose. For websites, the same idea applies: do not design “ugly” by default. Design against a specific dominant visual expectation. 

## 2. Separate expressive disorder from functional disorder

Anti-aesthetic web design should look unstable, raw, or uncomfortable where it helps expression, but it should not accidentally break core use.

Think in two layers:

### Expressive layer

This is where you can distort, clash, fragment, degrade, exaggerate, or make things feel “wrong.”

Examples:

* Strange spacing
* Unbalanced composition
* Overlapping blocks
* Harsh type combinations
* Low-resolution textures
* Awkward animation
* Intentionally ugly color palettes
* Aggressive borders
* Broken grid aesthetics
* Dense visual noise
* Unsettling empty space
* Anti-commercial copywriting

### Functional layer

This should remain reliable.

Keep:

* Links clickable
* Forms usable
* Navigation understandable enough
* Text readable when it matters
* Keyboard access intact
* Mobile layout functional
* Loading behavior reasonable
* Error states clear
* Critical information findable

A useful rule:

> Break the visual grammar, not the user’s ability to act.

This follows the paper’s distinction between deliberate anti-aesthetic choices and unintended model errors. In website terms, anti-aesthetic does not mean “the site is broken.” It means the site uses the language of brokenness deliberately. 

## 3. Build an anti-aesthetic concept map

The paper mentions dimensions such as lighting, color, realism, clarity, composition, detail, emotion, and prominence. These can be translated into web-design dimensions.

Use the following as a design bank.

## 4. Color: reject tasteful harmony

Most modern websites use soft gradients, carefully balanced palettes, safe neutrals, and pleasant accent colors. Anti-aesthetic websites can reject that.

Possible approaches:

### Clashing color

Use colors that fight each other.

Examples:

* Acid green on muddy brown
* Neon pink with dull gray
* Pure red beside default link blue
* Washed yellow with dirty purple
* Hospital green with corporate navy

Use clashing color when the site wants to feel aggressive, unstable, cheap, toxic, rebellious, or overstimulating.

### Dead color

Make the site feel drained.

Examples:

* Dusty beige
* Bruised gray
* Faded brown
* Mold green
* Overexposed off-white
* Low-saturation blue

Use dead color for themes like memory, boredom, bureaucracy, grief, institutional decay, or abandoned digital space.

### Over-bright color

Make the palette visually hostile.

Examples:

* Near-fluorescent backgrounds
* Warning-sign yellow
* Fake plastic gradients
* RGB colors without refinement
* Unsoftened red, green, and blue

This works well for satire, net art, fake commerce, conspiracy aesthetics, or intentionally irritating interfaces.

Practical rule: color can be ugly, but important text still needs enough contrast. Anti-aesthetic design can be uncomfortable without becoming unreadable.

## 5. Typography: make type feel human, cheap, unstable, or excessive

Modern websites often rely on clean sans-serif fonts, predictable hierarchy, and generous line spacing. Anti-aesthetic typography can challenge that.

### Use mismatched fonts

Pair fonts that are not supposed to go together.

Examples:

* Default serif body with futuristic display font
* Monospace headings with childish handwriting accents
* Elegant editorial serif with crude system UI buttons
* Times New Roman with pixel fonts
* Arial, Courier, and Georgia on the same page

This can create a feeling of amateurism, institutional weirdness, archival disorder, or early web energy.

### Break hierarchy

Instead of smooth heading levels, create awkward emphasis.

Examples:

* Tiny main titles
* Huge footnotes
* Body text that suddenly becomes oversized
* Captions that look more important than headlines
* Navigation that feels like a warning label

This works especially well when the site critiques authority, information overload, bureaucracy, or design conventions.

### Use ugly defaults intentionally

System fonts can be powerful.

Examples:

* Default blue underlined links
* Browser-style form inputs
* Unstyled buttons
* Plain HTML lists
* Monospace file-directory labels
* Default table styling

This creates a raw, anti-brand feeling. It says: “This was not polished for your comfort.”

## 6. Layout: resist the polished grid

Most websites use clean grids, centered containers, balanced cards, and predictable sections. Anti-aesthetic sites can feel misaligned, cramped, scattered, or overbuilt.

### Broken grid

Use visible misalignment.

Examples:

* Cards that do not share equal heights
* Text blocks pushed too far left or right
* Images placed slightly off-axis
* Uneven gutters
* Asymmetrical margins
* Sections that feel pasted together

This can suggest instability, refusal, collage, or handmade construction.

### Dense composition

Make the page feel overloaded.

Examples:

* Too many links
* Crowded sidebars
* Nested boxes
* Repeated labels
* Dense metadata
* Long scrolling lists
* Competing modules

This works for archives, activist sites, underground communities, glitch aesthetics, or satire of information systems.

### Empty, awkward composition

Use space that feels wrong rather than elegant.

Examples:

* A tiny object floating in a huge blank page
* Navigation isolated in a corner
* A paragraph stranded far below the fold
* Large dead zones
* A footer that appears too early

This can create loneliness, uncertainty, digital abandonment, or anti-commercial quiet.

The paper criticizes “sanitized reality,” where everything becomes polished, flawless, and universally beautiful. In web design, resisting that means allowing pages to feel partial, unresolved, strange, or emotionally specific. 

## 7. Imagery: avoid generic beauty

Even though your question is about websites rather than AI images, website imagery still matters.

Anti-aesthetic imagery can include:

* Blurry photos
* Harsh flash photography
* Cropped faces or objects
* Overcompressed JPEGs
* Screenshots with artifacts
* Scanned paper
* Dust, stains, photocopy textures
* Boring objects
* Empty rooms
* Bad stock-photo framing used intentionally
* Low-light images
* Overexposed images
* Mundane documentary photos

But the key is intentional framing. A blurry photo should not look like the designer forgot to export it correctly. It should feel like blur belongs to the concept.

Good uses:

* A grief archive using faded, low-contrast photos
* A punk event site using harsh flash and grain
* A critique of corporate wellness using dead-eyed stock imagery
* A noise music site using overcompressed, unreadable images
* A fictional bureaucracy site using ID-card-like photography

Bad uses:

* Random blurry hero image because no better image was available
* Low-res images on a luxury site with no conceptual reason
* Compression artifacts that make product details impossible to see

## 8. Motion: make interaction feel strange, not merely annoying

Modern UI motion is usually smooth, helpful, and invisible. Anti-aesthetic motion can be awkward, delayed, jittery, mechanical, or excessive.

Possible techniques:

* Slightly delayed hover states
* Janky marquee text
* Blinking cursors
* Abrupt transitions
* Glitchy scroll effects
* Mechanical loading animations
* Elements that tremble subtly
* Menus that open too dramatically
* Cursor trails
* Repeating GIF-like movement
* Intentionally stiff animation

Use motion to create personality, not punishment.

Avoid:

* Flashing that could harm photosensitive users
* Scroll hijacking that traps users
* Animations that prevent reading
* Motion that cannot be reduced
* Delays on critical actions like checkout, login, or form submission

Offer a reduced-motion version. Anti-aesthetic does not require ignoring accessibility.

## 9. Navigation: let it be weird, but not impossible

Navigation can be unconventional, but users still need a way through.

Anti-aesthetic navigation patterns:

* Directory-style navigation
* Site map as homepage
* Text-only navigation
* Floating labels
* Fake desktop folders
* Broken museum archive layout
* Marginal notes as links
* Repeated links in different formats
* “Wrong” labels that still make sense in context

For example, instead of:

* Home
* About
* Work
* Contact

An anti-aesthetic site might use:

* Entrance
* Evidence
* Noise
* Receipts
* Exit

This creates a world. But each label should still be learnable.

A good test: after 30 seconds, can a user roughly understand where they are and how to keep moving? If yes, the weirdness is probably productive. If no, it may just be bad navigation.

## 10. Content voice: reject smooth brand language

Most commercial websites sound frictionless:

> We create meaningful digital experiences for modern teams.

Anti-aesthetic websites can use language that is more blunt, awkward, paranoid, poetic, bureaucratic, absurd, exhausted, or emotionally specific.

Examples:

### Bureaucratic anti-aesthetic

> Your request has been noticed.
> It may or may not be processed.
> Do not refresh the page emotionally.

### Deadpan anti-commercial

> This is a website.
> It contains work, prices, and several attempts at credibility.

### Raw personal

> I made this because the polished version felt dishonest.

### Archival

> Folder 03 contains failed posters, scanned notes, and one image that should not have survived.

The paper argues against forcing all outputs into positivity and conventional appeal. For websites, this means the copy does not always need to sound helpful, upbeat, optimized, or friendly. It can be uncomfortable when the project calls for that. 

## 11. Use negative emotion carefully and legitimately

The paper makes an important point: negative emotions are not automatically unsafe or bad. They can support critique, reflection, learning, empathy, and artistic expression. 

For web design, this means you can design for:

* Sadness
* Anxiety
* Decay
* Anger
* Emptiness
* Dread
* Alienation
* Exhaustion
* Disgust
* Awkwardness

But the site should avoid manipulating users in harmful ways.

Good:

* A memorial site that feels quiet, dim, and heavy
* A climate project using visual decay and polluted colors
* A labor-rights site using oppressive layouts to show bureaucracy
* A horror publication using darkness, unease, and fragmented navigation

Bad:

* A checkout page that uses anxiety to force purchases
* A health site that uses dread to increase engagement
* A donation page that overwhelms users with guilt
* A service page that hides cancellation through hostile design

Anti-aesthetic design is not the same as dark patterns.

## 12. Preserve accessibility even when aesthetics are hostile

This is the part designers often get wrong.

Anti-aesthetic websites can be visually uncomfortable, but they should still be accessible. You can make something ugly without making it exclusionary.

Keep:

* Semantic HTML
* Keyboard navigation
* Visible focus states
* Alt text
* Sufficient contrast for important text
* Captions or transcripts for media
* Reduced motion support
* Resizable text
* Clear form labels
* Skip links
* Error messages that explain what happened

You can create a second “plain mode” or “reader mode” if the design is intentionally noisy.

For example:

* “Reduce noise”
* “Plain text version”
* “Low motion”
* “Readable mode”
* “Disable effects”

That does not weaken the anti-aesthetic concept. It strengthens it, because the design becomes intentional rather than careless.

## 13. Treat anti-aesthetic design as a spectrum

The paper argues for “wide-spectrum aesthetics,” not replacing conventional beauty with one new ugly standard. 

For websites, avoid turning anti-aesthetic into another trend where every site uses the same:

* Acid green
* Pixel font
* Glitch effect
* Brutalist grid
* Default Times New Roman
* Marquee text
* Harsh black-and-white layout

Instead, define intensity levels.

### Level 1: Slight friction

Useful for portfolios, editorials, small brands.

* Odd font pairing
* Slightly awkward spacing
* Raw photography
* Muted colors
* Minimal animation

### Level 2: Clear anti-polish

Useful for art, music, fashion, criticism, culture.

* Broken grid
* Clashing colors
* Visible compression
* Strange navigation
* Dense information
* Rough UI elements

### Level 3: Full anti-aesthetic world

Useful for net art, experimental projects, political critique, fictional interfaces.

* Chaotic layout
* Surreal navigation
* Fragmented copy
* Harsh motion
* Heavy texture
* Distorted hierarchy
* Strong emotional tone

### Level 4: Controlled hostile interface

Use rarely.

* Confusing structure
* Aggressive color
* Intentionally unpleasant sound or motion
* Abrupt changes
* Unstable page behavior

This level is only appropriate for art projects, games, critique, or experimental storytelling. It is usually wrong for public services, commerce, education, health, or anything task-critical.

## 14. Match anti-aesthetic style to website type

### Portfolio

Use anti-aesthetics to show personality, not to hide the work.

Good choices:

* Weird index page
* Raw thumbnails
* Strange typography
* Unpolished project notes
* Experimental hover states

Keep case studies readable.

### Art or music site

You can go much further.

Good choices:

* Glitch, noise, grain
* Nonlinear navigation
* Unstable layout
* Low-light visuals
* Fragmented text
* Sound-reactive elements

Still provide basic event, purchase, or contact info clearly.

### Activist or critique site

Anti-aesthetics can challenge sanitized communication.

Good choices:

* Harsh contrast
* Documentary imagery
* Dense evidence layouts
* Bureaucratic visual language
* Broken institutional motifs
* Aggressive typography

Keep calls to action clear.

### Fashion or cultural brand

Anti-aesthetic design can resist luxury minimalism.

Good choices:

* Bad taste references
* Clashing product photography
* Rough layout
* Overprinted text
* Awkward crops
* Strange product descriptions

Keep sizing, pricing, shipping, and checkout usable.

### SaaS or product site

Use lightly unless the product itself is experimental.

Good choices:

* Less polished illustrations
* Honest copy
* Raw screenshots
* Anti-template layout
* Slightly strange brand system

Do not damage trust around pricing, security, onboarding, or support.

## 15. Design with “controlled wrongness”

A practical framework:

For every anti-aesthetic choice, answer three questions.

### 1. What norm does this violate?

Examples:

* Good spacing
* Pleasant color
* Clean hierarchy
* Smooth animation
* Positive tone
* Professional photography
* Minimal navigation

### 2. Why does violating it help?

Examples:

* Makes the brand feel less corporate
* Communicates decay
* Creates discomfort
* Shows amateur sincerity
* Critiques luxury polish
* Makes the page feel archival
* Reflects emotional disorder

### 3. What remains stable?

Examples:

* Main text remains readable
* Checkout remains normal
* Contact remains visible
* Keyboard access remains intact
* Mobile layout still works
* Motion can be reduced

This prevents the site from becoming randomly bad.

## 16. Evaluation: do not judge it by “prettiness”

The paper’s central warning is that reward systems often penalize anti-aesthetic outputs even when they match the prompt better than conventionally beautiful outputs. 

For websites, do not evaluate anti-aesthetic design with only questions like:

* Is it beautiful?
* Is it clean?
* Is it modern?
* Would most people like it?
* Does it look premium?

Instead evaluate:

* Does the site match the intended emotional tone?
* Is the ugliness deliberate and coherent?
* Does the visual discomfort support the concept?
* Can users still complete important actions?
* Does the site avoid accidental accessibility failures?
* Does it resist generic design in a meaningful way?
* Does it create a memorable identity?
* Does it avoid becoming just another trendy brutalist template?

A good anti-aesthetic site may score low on mainstream polish but high on expression, memorability, critique, and fit.

## 17. Anti-aesthetic website checklist

Use this before shipping.

### Intent

* The site has a clear reason to reject conventional beauty.
* The anti-aesthetic choices connect to the message, audience, or concept.
* The site is not ugly just because the designer skipped refinement.

### Visual system

* Color choices are intentionally awkward, dead, harsh, or unstable.
* Typography supports the mood.
* Layout breaks expectations without destroying basic orientation.
* Imagery avoids generic polish.
* Motion adds tone without blocking use.

### Function

* Users can navigate.
* Links and buttons work normally.
* Important text is readable.
* Forms are usable.
* Mobile is not accidentally broken.
* Loading and error states are understandable.

### Accessibility

* Keyboard navigation works.
* Focus states are visible.
* Important contrast is acceptable.
* Motion can be reduced.
* Images have useful alt text.
* A plain or reduced-noise mode exists if the site is very intense.

### Conceptual strength

* The site resists a specific visual norm.
* The discomfort is meaningful.
* The design does not collapse safety, comfort, and beauty into one thing.
* The site allows emotional plurality, including negative or strange moods.
* It does not confuse deliberate anti-aesthetics with accidental defects.

## 18. A simple design recipe

Here is a practical process.

1. Choose the target norm: polished SaaS, luxury minimalism, cheerful wellness, corporate optimism, clean editorial, social media beauty, etc.

2. Choose the emotional direction: anxious, dead, absurd, angry, haunted, cheap, bureaucratic, nostalgic, noisy, lonely.

3. Pick 3 to 5 anti-aesthetic dimensions:

   * Color clash
   * Low clarity
   * Bad spacing
   * Ugly typography
   * Dead imagery
   * Dense composition
   * Negative emotion
   * Awkward motion
   * Broken grid
   * Harsh contrast

4. Decide what must stay functional:

   * Navigation
   * Primary CTA
   * Reading
   * Purchase flow
   * Contact
   * Search
   * Accessibility

5. Prototype two versions:

   * Conventional version
   * Anti-aesthetic version

6. Compare by intent, not beauty:

   * Which one says the right thing?
   * Which one feels more specific?
   * Which one better avoids generic polish?
   * Which one still lets users act?

## 19. Example design directions

### “Corporate rot”

A site that looks like a decaying internal company portal.

* Gray-green palette
* Default form fields
* Dense tables
* Fake policy language
* Slightly misaligned cards
* Bureaucratic labels
* Deadpan copy
* Tiny footer notes
* Occasional broken-looking stamps

Best for: critique, fiction, art, institutional satire.

### “Low-budget sincerity”

A site that rejects polished branding and feels handmade.

* Default links
* Simple HTML structure
* Scanned sketches
* Unbalanced spacing
* Personal notes
* System fonts
* No glossy photography
* Plain buttons

Best for: artists, zines, small studios, independent projects.

### “Digital grief”

A quiet, emotionally heavy website.

* Dim colors
* Low contrast used carefully
* Slow transitions
* Faded images
* Sparse layout
* Large empty areas
* Fragmented text
* Archival navigation

Best for: memorials, essays, reflective projects, documentary work.

### “Overstimulated trash web”

A noisy, aggressive interface.

* Clashing neon
* Repeating banners
* Marquee text
* Dense links
* Animated stickers
* Low-res graphics
* Chaotic hierarchy
* Excessive borders

Best for: music, net art, underground culture, satire.

Use this carefully. It can become gimmicky fast.

# Bottom line

An anti-aesthetic website should not be “bad design.” It should be **deliberate design against dominant beauty norms**.

The strongest version keeps the paper’s main principle: user intent and expressive range matter more than universal prettiness. A good anti-aesthetic website may be ugly, dark, awkward, noisy, or emotionally negative, but it should still be intentional, coherent, and usable. It should widen what websites are allowed to feel like, rather than replacing one rigid aesthetic standard with another.
